# 조직 조류 · People Flow Intelligence

자금 조류(money flow)의 **동형(isomorphic) HR 버전.** 시장의 약신호를 읽던 문법을
_조직의 약신호_(이탈·번아웃·몰입저하)로 옮겼다. 핵심은 **신호 판정 커널을 그대로
재사용**한다는 것 — `dashboard/signal_engine.js`의 `zlast`·`tierOf`는 도메인 불변이라
HR 시계열에 글자 그대로 다시 쓴다. "동형"은 수사가 아니라 코드다.

```bash
python hr/hr_synth.py        # 합성 데이터 생성 → hr/hr_data.js (window.HR_DATA)
node   hr/hr_engine.test.js  # HR 신호 엔진 단위 테스트
open   hr/index.html         # 조직 조류 대시보드 (브라우저로 바로)
```

**라이브:** https://foolpoet44.github.io/money_flow_2026/hr/ (GitHub Pages 발행)

## 왜 이 도메인인가

머니플로 백테스트는 공개·후행 데이터엔 투자 엣지가 없음을 보였다(`backtest/`). 반면
HR은 **사용자가 실제 도메인 엣지를 가진** 곳이고, 자동화가 곧 수입 레버(컨설팅·사내
도구·HR-tech)를 직접 움직인다. 같은 엔진, 엣지가 있는 무대로의 이동.

## 동형 매핑 (money flow ↔ HR)

| 머니플로                        | 조직 조류                    | 의미                    |
| ------------------------------- | ---------------------------- | ----------------------- |
| 종목/지수                       | 팀/전사                      | 분석 단위               |
| 외국인·기관 순매수              | 초과근무·협업·eNPS           | 약신호 지표             |
| 30거래일                        | 12주                         | 자기참조 시계열         |
| `zlast()`                       | `zlast()` _(동일 커널)_      | 자기 기준선 대비 극단도 |
| confluence(수급·가격 정렬)      | confluence(위험 약신호 정렬) | 단일지표 불신           |
| `tierOf()` Watch/Alert/Critical | `tierOf()` 동일              | 등급 판정               |
| "외국인 단독 순매도는 노이즈"   | "초과근무만 ↑는 노이즈"      | 같은 철학               |

핵심 교훈도 그대로다 — **같은 z 크기여도 정렬된 약신호의 수가 등급을 가른다.** 백엔드팀과
프론트팀이 동일 z=1.9여도, 3중 정렬(초과근무↑·협업↓·eNPS↓)인 백엔드팀은 Critical,
단일(초과근무↑)인 프론트팀은 Alert다.

## 데이터 계약 (hr_data)

```jsonc
{
  "as_of": "2026-W24",
  "weeks": ["2026-W13", ...],          // length 12
  "org": {
    "attrition_pressure": [...],        // 이탈압력 지수(↑=위험). 조직 tide.
    "engagement": [...], "headcount_net": [...]
  },
  "teams": [
    { "id":"T01","name":"백엔드팀","size":8,
      "overtime":[...],                 // 주당 초과근무h (↑=번아웃 위험)
      "collab":[...],                   // 협업활동지수 0~100 (↓=몰입저하 위험)
      "enps":[...] }                    // 펄스 eNPS −100~+100 (↓=정서약화 위험)
  ]
}
```

각 지표는 엔진에서 **위험=양(+)** 방향으로 정규화된다(협업·eNPS는 부호 반전). 위험
방향으로 `|z|≥1.0`인 지표가 몇 개 정렬되는지가 confluence다.

## 윤리 가드레일 (반드시 지킬 것)

이 도구는 **감시 도구가 아니다.** 머니플로의 "거짓 데이터 금지"에 대응하는 HR판 원칙:

1. **분석 단위는 팀.** 개인 워치리스트를 만들지 않는다. 개인 이탈예측은 기술적으로
   가능하나 신뢰·법(개인정보보호법)·편향 문제로 이 데모에서 의도적으로 배제했다.
2. **자기참조 판정.** 팀을 서로 비교(횡단면)하지 않고 각 팀 자신의 기준선과 비교한다 —
   사람을 줄 세우지 않는다. 횡단면 편향은 피플 애널리틱스의 1번 오류다.
3. **지원적 관점.** 목적은 처벌이 아니라 경영진의 **지원적 주의**(번아웃 케어, 1:1,
   업무재분배)다. 신호는 트리거가 아니라 대화의 시작점이다.
4. **합성 데이터.** 본 데모는 전부 합성이다. 실 데이터 연결 시 익명화·집계·동의·접근통제가
   전제다.

## 실데이터 어댑터 (`hr_adapter.py`)

합성 생성기 자리에 끼우는 CSV 어댑터. 현실적인 원천(HRIS·근태·협업로그·펄스서베이의
CSV 내보내기)을 **팀 단위로 집계**해 같은 hr_data 계약을 만든다 — 머니플로가 collector만
갈아끼웠던 것과 동형. 엔진·대시보드는 한 줄도 안 바뀐다.

```bash
python hr/hr_adapter.py --make-sample   # (데모) 합성 팀 시계열을 개인 단위 CSV로 역집계
python hr/hr_adapter.py                  # CSV → hr_data.json + hr/hr_data.js
```

**입력 CSV 스키마** (`hr/sample_data/`에 샘플 동봉):

| 파일            | 컬럼                            | 비고                 |
| --------------- | ------------------------------- | -------------------- |
| `roster.csv`    | person_id, team                 | 사람→팀 매핑         |
| `overtime.csv`  | person_id, week, overtime_hours | 근태/근무시간        |
| `collab.csv`    | person_id, week, collab_score   | 협업로그(0~100)      |
| `enps.csv`      | person_id, week, enps           | 펄스서베이(−100~100) |
| `headcount.csv` | week, net                       | 주간 순증감          |

**코드로 강제하는 윤리:**

- **k-익명성** — 팀 인원이 `MIN_TEAM_SIZE`(기본 5) 미만이면 집계에서 제외(개인 식별 방지).
- **개인 식별자 미보존** — 집계 후 person_id를 출력에 남기지 않는다.
- 개인 단위 점수·랭킹을 만들지 않는다.

검증: 합성 팀 시계열 → 개인 CSV(32명, 멤버 오프셋 합=0) → 어댑터 재집계 → **동일 신호
그라데이션** 복원(왕복 정합성). 실제 HRIS 연결 시엔 익명화·동의·접근통제가 전제다.
